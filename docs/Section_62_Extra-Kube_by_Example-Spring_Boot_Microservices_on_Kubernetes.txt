## Overview of Spring Boot Microservices

### Brewery Spring Boot Microservices
- These Spring Boot Microservices originally developed for my course Spring Boot Microservices with Spring Cloud Beginner to Guru
    - In this course, we are not focusing on details of the services, but deployment on Kubernetes
- Three microservcies to used to mimic the operation of a brewery
- Data is persisted to MySQL via JPA/Hibernate
- JMS is used for messaging between services
- Beer Inventory Service - Holds information about invetory levels.
    - REST API used to get inventory information
    - JMS Messaging used to increase inventory from brewing, decrease inventory from order allocation
- Beer Service - holds information about beers brewed, REST API for Beer CRUD operations
    - JMS Messaging to ‘brew’ beer when inventory is low
    - Scheduled job to ‘brew’ beer
- Beer Order Service - Holds information about Beer Orders
    - REST API for Beer Order Operations
    - JMS Messaging to ‘allocate’ beer orders
    - Tasting Room Service - scheduled job to ‘order’ beers, which decreases inventory, which triggers brewing
- Important to understand Beer Order Service will generate orders, request allocations from Inventory Service (reducing inventory), which triggers Beer Service to ‘brew’ beer (adding to inventory)
- Inventory Failover Service - Service to provide a positive response if inventory service is not available.
    - Always returns an on-hand quantity of 999

### Original Configuration
*Deve ser transformado em um diagrama/fluxo usando Mermaid ou gerar um imagem representando o fluxo de configuração.*
- API Gateway ↔ Eureka
- API Gateway ↔ Beer Service ↔ MySQL
- API Gateway ↔ Order Service ↔ MySQL
- API Gateway ↔ Inventory Service ↔ MySQL
- API Gateway ↔ Inventory Failover
- Config Service ↔ Beer Service
- Config Service ↔ Eureka
- Config Service ↔ Inventory Service
- Beer Service ↔ ActiveMQ
- Inventory Service ↔ ActiveMQ

### Changes for Kubernetes
- **Eureka:** is not used. Kubernetes is used for service discovery.
- **Spring Cloud Config:** Not Used. Kubernetes used to manage environment properties.
- **Single MySQL Insntace:** One instance used. Just for simplicity, a production deployment should have independent MySQL database instances.
- **Single GitHub Repo:** Again for simplicity. Microservices typically would have independent source code repositories.
