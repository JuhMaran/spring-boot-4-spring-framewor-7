### Maven Standard Directory Layout
- Maven Standard Directory Layout - Established by Maven with the 2.0 release in 2005
- Industry Standard for Java Projects
- Also used by other Java build tools, such as Gradle
- Project Directory:
    - pom.xml - Maven Build File with project metadata, dependencies, build plugins, etc.
    - [README.md](http://README.md) - Markdown file with project information, also shown on root page of GitHub
    - /target - Project build artifacts
    - /src - All source files for building the project
        - /main - Application Source Files
            - /java - Java Source Files
            - /resources - Non Java source files, such as property files or images
        - /test - Application Source Files
            - /java - Java Source Files for Unit Tests
            - /resources - Non Java source files, such as property files or images for Unit Tests

```
project/
 ├── .gitignore
 ├── pom.xml                      # 
 ├── /src
 |    ├── /main
 |    |    ├── /java
 |    |    └── /resources
 |    └── /test
 |         ├── /java
 |         └── /resources      
 └── README.md
```

### Spring Boot Specific Files
- /src/main/resources/application.properties - Key Value properties file
- /src/main/resources/application.yaml- YAML formatted properties file
- /src/test/resources/application.properties - Key Value properties file for Unit Tests
- /src/tes/resources/application.yaml- YAML formatted properties file for Unit Tests
- /src/main/resources/static - Static Web files, served from root url (images, javascript, etc.)
- /src/main/resources/templates - Web Application Template files
- /src/main/resources/db/migration - Database Migration Scripts

### Spring Boot Default Classes
- In the default package of your project (com.myproject.example) is the Spring Boot Main Class
    - The Main Class is annotated with `@SpringBootApplication`
    - The main method is executed at run time to start the Spring Boot Application
- Important - Spring Boot will scan for components in the default package and all child packages
    - Components not under the default package will NOT be picked up
- Projects created using the Spring Initializer will have a default test class for the application
    - Simple test to verify Spring is able to start without errors.

### Spring Boot Main Class Example
```java
@SpringBootApplication
public class DemoApplication {
  public static void main(String[] args) {
    SpringApplication.run(DemoApplication.class, args);
  }
}
```

### Spring Boot Test Example
```java

  @Test
  void contextLoads() { }
}
```

- A common pattern is to organize by type/function

### What is MVC?
- **MVC** is a common design pattern for GUI and Web Applications
    - Used in many programming languages
- **M** - Model
- **V** - View
- **C** - Controller

### MVC
**Descrição da imagem:** Controller, Model e View, estão dentro de uma caixa pontilha do Spring MVC. O Client Request envia uma requisição para a aplicação, o Controller recebe, passando pelo Model e View. Sendo que a View retorna para o Client Request.
E temos uma comunicação entre Controller e Biz Logic
- Client Request → Controller → Model → View → Client Request

### Spring MVC
- **MODEL:** Simple POJO with collection of properties which may or may not be used by the view
- **VIEW:** Data as requested by the client. Implemented with JSP, Thymeleaf, Jackson, etc.
    - Can be HTML, JSON, XML, text, etc.
- **CONTROLLER:** Java class implemented to handle request mapping. Should be a “traffic cop” and contain minimal business logic. Typically works in conjunction with a service, which contains business logic.
**Descrição da imagem:** Client Request envia a requisição para Dispatcher Servlet que envia para o Controller. O Controller envia para o Service (que detém a lógica) e busca/envia no Banco de Dados (Spring Data JPA) as informações requisitadas. O Controller recebe a informação do Service, que segue o fluxo pelo Model (POJO) e depois para a View. A View retornar a requisição para Dispatcher Servlet que envia para o Client Request.
